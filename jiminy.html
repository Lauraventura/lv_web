Tick-track allows you to stablish GPS located alarms so that whenever you run by a place where you have some unfinished business you can always recall it.

Research

This project began with a research about people's painpoints surrounding memory issues. The main problem is not about anything you would note in your agenda, it's about all those things that can't be adjusted to timetables, just because you don't know the exact time you'll be able to do them, or beacuse you need to be in a specific place to fulfill a task.

At this point, I began to look into all the resources people used to remind themselves about things. They went from notes in the palm of their hand, to combining 10 apps hoping to look at one of them at the correct time.

Concept

According to users, the hardest reminders to set are those where time is uncertain. If we set an alarm at 5PM expecting to be home at that time but the train runs late and we am not there, the alarm will probably be useless.

The correct metric system in this case would be location, not time. That's the focus of this project: setting GPS located alarms to optimize our time management.

IxD

I tried to figure out when would a user actually open the app. That would be in two situations mainly: to set a new alarm, and to find the best way to get to the nearest one. That's why the main screen contains a list of alarms sorted by proximity, and a menu with a big blue button to create a new one.

I also had to design the messages and flows for the alarms, as it is 50% of the user experience of my app.

UI

This project is not about flashy colors and trendy images. It had to be super functional and clean. I wanted to set a very clear hierarchy to guide the user straigt to the point.

Grayish-Blue was a very calm color that allowed me to create great contrast with white without going flashy.
